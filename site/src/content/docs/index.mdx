<section class="ccr-section">
  <h2 class="ccr-h2">60-second setup</h2>
  <p class="ccr-sub">Drop one workflow file in your repo, add your API key as a secret — reviews run automatically on every pull request.</p>

```yaml title=".github/workflows/vor.yml"
name: Vor
on:
  pull_request:
    types: [opened, synchronize, reopened, ready_for_review]
  workflow_dispatch:
    inputs:
      pr_number:
        description: 'PR number to review'
        required: true
      model:
        description: 'Model to use'
        required: false
        default: 'claude-sonnet-4-6'
      dry_run:
        description: 'Log review instead of posting comments'
        required: false
        default: 'false'
        type: boolean

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - uses: driches/vor@v0
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          pr_number: ${{ inputs.pr_number }}
          model: ${{ inputs.model || 'claude-sonnet-4-6' }}
          dry_run: ${{ inputs.dry_run || 'false' }}
```

  <p class="ccr-step">Prefer OpenAI? Swap in <code>openai_api_key</code> and set <code>model: gpt-4.1</code> — the provider is inferred from the model id. You can also re-run or review any PR on demand via <strong>Actions → Vor → Run workflow</strong>.</p>

  <h2 class="ccr-h2">Why this works when “ask the AI to review the PR” doesn’t</h2>
  <div class="ccr-why">
    <div class="ccr-why__card">
      <h3>Output is actionable, not prose</h3>
      <p>
        The agent has no free-text output channel. Findings can only surface through the validated{' '}
        <code>post_inline_comment</code> tool.
      </p>
    </div>
    <div class="ccr-why__card">
      <h3>Comments land inline</h3>
      <p>
        Reviews post via <code>pulls.createReview</code> with a structured <code>comments[]</code>{' '}
        array — path, line, side, and suggestion blocks.
      </p>
    </div>
    <div class="ccr-why__card">
      <h3>No hallucinated lines</h3>
      <p>
        The validator rejects any line outside the real diff and returns the valid ranges as a hint,
        so the agent self-corrects.
      </p>
    </div>
