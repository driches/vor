# Vendored OCR assets

The `image_ocr` scanner and the `describe_image_at_ref` tool run
[`tesseract.js`](https://github.com/naptha/tesseract.js) fully offline — no CDN
fetch, no `npm install` — by reading the language model and WASM core from this
directory. The shipped Action runs `node dist/index.js` with no `node_modules`,
so everything the OCR worker needs is committed here.

Contents:

- `eng.traineddata` — English LSTM model, plain (not gzipped), from
  [`tessdata_fast`](https://github.com/tesseract-ocr/tessdata_fast). Loaded via
  `langPath`/`cachePath` with `gzip: false`.
- `tesseract-core.wasm`, `tesseract-core-simd.wasm`,
  `tesseract-core-relaxedsimd.wasm` — the three non-LSTM emscripten cores
  `tesseract.js`'s node `getCore` can select at runtime by CPU feature-detect
  (base / SIMD / relaxed-SIMD). The matching `.js` loaders are bundled into the
  worker (below); each reads its `.wasm` sibling from this directory via
  emscripten's `locateFile`.
- `tesseract-worker.cjs` — **build output**, regenerated by `npm run build`
  (`scripts/build.ts`) and guarded by `verify-dist`. It is the
  `tesseract.js`/`tesseract.js-core` `worker_threads` entry bundled into one
  self-contained CommonJS file (`.cjs` so node runs it as CJS despite the root
  package's `"type": "module"`). Do not edit by hand.

`tesseract.js`'s main thread is bundled into `dist/index.js`;
`src/ocr/recognize.ts` resolves this directory relative to the built bundle
(`<bundle>/../assets/ocr`), overridable with the `VOR_OCR_ASSETS_DIR` env var.

When these assets (or the `tesseract.js` runtime) are absent — e.g. a local
checkout that hasn't run `npm run build` — OCR degrades gracefully: `image_ocr`
and the OCR half of `describe_image_at_ref` return no text rather than failing
the review. The vision half of `describe_image_at_ref` (visual understanding)
needs no local assets — it calls the configured model.

## Updating

After bumping `tesseract.js` / `tesseract.js-core`, re-vendor the cores and
rebuild the worker:

```sh
cp node_modules/tesseract.js-core/tesseract-core.wasm \
   node_modules/tesseract.js-core/tesseract-core-simd.wasm \
   node_modules/tesseract.js-core/tesseract-core-relaxedsimd.wasm assets/ocr/
npm run build   # regenerates assets/ocr/tesseract-worker.cjs
```

The `eng.traineddata` model is version-independent; refresh it from the
`tessdata_fast` release only if you want a newer model.
